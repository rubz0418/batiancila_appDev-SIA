Description of the repository: 
This repository contains compiled laboratory activities, projects, and exercises for the subjects Application Development and Emerging Technologies (AppDev) and System Integration and Architecture 2 (SIA 2). The activities focus on emerging technologies, software design, Laravel development, database integration, API implementation, form validation, and system architecture concepts.

Application Development and Emerging Technology Activities
- Activity 1 : Emerging Technologies Exploration
               This activity focuses on exploring Artificial Intelligence and its real-world applications. It discusses how AI works, its                     advantages and disadvantages, its impact on society and industries, and future predictions about how AI may continue to develop                in the next 5–10 years. The activity helps students understand the importance of emerging technologies in modern life and                      future careers.

- Activity 2 : SecureExam Online Examination System
               This activity presents the proposed system called SecureExam, a web-based platform for managing online examinations. It
               explains the system’s purpose, users, and major functions such as exam creation, grading, monitoring, and communication
               features. The activity also includes a UML Use Case Diagram to show how Admins, Teachers, and Students interact with the system.

- Activity 4 : Design Your Own System Using UML
               This activity involves designing the SecureExam using UML diagrams and applying the Software Development Life Cycle (SDLC). It                 includes creating Use Case and Activity Diagrams, explaining each SDLC phase, and reflecting on the importance of system                       planning and UML in software development. The activity helps students practice system analysis, design, and development
               planning skills.

- Activity 5 : This activity involves building a Laravel-based web application that integrates authentication, internal APIs, and external                    APIs into one system. Users can register and log in securely, view dashboard information, access user data through a custom                    API, and display data fetched from a public API. The activity demonstrates basic system integration and API implementation in                  Laravel.


System Integration and Architecture 2 Activities:
- Activity 2 : Laravel Mini System
               This activity focuses on creating a simple Laravel mini-system using routes, controllers, Blade templates, and data passing.
               Students are required to build a unique system with at least two routes: one for displaying a list of items and another for
               showing item details. The activity also includes creating reusable layouts and displaying dynamic data using Blade templating.
- Activity 3 : Laravel CRUD System with Database
               This activity involves developing a complete Laravel CRUD (Create, Read, Update, Delete) system connected to a MySQL database.                 Students must configure the database, create migrations, and implement CRUD functionalities using controllers and Blade
               templates. The system should allow users to add, view, edit, and delete records dynamically from the database.
- Activity 4 – Laravel Form Handling & Validation
               This activity is about creating a Laravel form system with proper input validation and error handling. Students need to design                 a form with different input fields, process form submissions using controllers, and apply validation rules to ensure correct                   user input. Validation errors must be displayed clearly, and the form should retain old input values after submission.
-Activity 5 - Related to AppDev activity 5
              This activity focuses on developing and integrating a Laravel application with APIs. It covers authentication setup using                      Laravel Breeze, creating a REST API, consuming data from a public API, and combining all components into a functional dashboard                system. The activity helps students understand basic system integration concepts and API implementation.


HINUNANGAN SHOP VISIT
Hinunangan Shop Visit: Shop Visit Log System is a Laravel-based system that combines Activities 2, 3, and 4 by showing shop visit records, saving them in a MySQL database, and allowing users to add, view, update, delete, search, and upload images for visits. It also includes a validated request form to demonstrate Laravel form handling, CSRF protection, error messages, old input, and success messages.
